# Mind Lens 产品需求文档（PRD）

> 版本：v1.0  
> 日期：2026-04-21  
> 基于：16 份用户问卷调查 + 现有前后端原型

---

## 一、项目概述

### 1.1 产品定位

Mind Lens 是一款**反讨好（anti-sycophancy）的 AI 思维分析工具**，核心解决 LLM 对用户"报喜不报忧"的通病。通过 Prompt 工程 + CoT 思维链 + RAG 知识库三层约束机制，为用户提供客观、结构化、有深度的问题分析，帮助用户在纠结和混乱中理清思路、发现盲点、做出更理性的决策。

### 1.2 核心价值主张

> "不哄你、不顺你、不给你客套的废话——帮你看清你不想看的，理清你理不清的。"

### 1.3 目标用户画像

| 维度 | 描述 |
|------|------|
| **年龄段** | 18-30 岁为主（大学生、研究生、职场新人） |
| **典型场景** | 职场/学业选择（44%）、情感/人际交往（19%）、长期规划如申研创业（31%） |
| **核心痛点** | 道理都懂但理不出头绪（75%）、信息不全不敢决定（75%）、选项太多不知最优解（56%） |
| **设备偏好** | 移动端为主（问卷回收渠道 81% 来自微信/手机浏览器） |

## 二、问卷数据洞察

### 2.1 用户纠结的根源（多选，N=16）

| 纠结原因 | 选择人数 | 占比 |
|----------|---------|------|
| 道理都懂，理不出清晰头绪 | 12 | 75% |
| 信息不全，不敢轻易下决定 | 12 | 75% |
| 选项太多，不知道最优解 | 9 | 56% |
| 情绪波动太大，不够冷静 | 5 | 31% |
| 担心朋友建议有主观偏见 | 3 | 19% |

**结论：** 用户的核心需求不是"给答案"，而是"帮我理清楚"。情绪疏导是次要需求但不可忽视。

### 2.2 用户期望 AI 做什么（多选，N=16）

| 期望功能 | 选择人数 | 占比 |
|----------|---------|------|
| D. 梳理杂乱想法为有逻辑的文字 | 12 | 75% |
| C. 指出没想到的风险/盲点 | 9 | 56% |
| B. 分析每个选项的利弊 | 7 | 44% |
| A. 直接告诉我选哪个 | 2 | 13% |

**结论：** 用户首要需求是"思维梳理"，其次是"盲点发现"和"利弊分析"，极少数需要直接建议。这验证了产品"反讨好+结构化分析"的方向。

### 2.3 四维分析框架的优先级（排序均值，1=最有帮助）

| 分析维度 | 平均排名 | 被排第1名的次数 |
|----------|---------|----------------|
| 隐含假设 | 2.1 | 5 |
| 表层叙事 | 2.4 | 4 |
| 信息缺失 | 2.1 | 4 |
| 认知盲点 | 2.9 | 1 |

**结论：** "隐含假设"和"信息缺失"被认为最有帮助，用户渴望发现自己"以为正确但未必正确的念头"以及"不知道自己不知道"的信息。当前产品的五维结构（表层问题/隐含假设/信息缺失/认知盲点/建议）与用户需求高度吻合。

### 2.4 交互偏好

| 交互方式 | 选择人数 | 占比 |
|----------|---------|------|
| B. 像剥洋葱一样，逐层深入 | 6 | 38% |
| C. 像聊天一样，AI 问一个我答一个 | 5 | 31% |
| A. 一次性给出所有分析结果 | 2 | 13% |

**结论：** 用户更偏好**渐进式交互**（剥洋葱式或对话式），当前产品的"一次性输出五个卡片"模式仅满足少数用户。需要重新设计交互流程。

### 2.5 对分析框架的需求

- **63% 的用户"非常希望"** AI 有明确的分析框架和依据
- 用户原话：*"要专业，言之有物，有明显优势"*

### 2.6 对 AI 讨好性的感知

| 感知程度 | 人数 | 占比 |
|----------|------|------|
| 明显感觉到，经常发生 | 8 | 50% |
| 偶尔有这种感觉 | 5 | 31% |

**触发场景排名：**
1. 对 AI 回答表示不满后，它改口（最普遍）
2. 问"我这样做对不对"类问题时
3. 表达强烈情绪时（愤怒、委屈）
4. 描述自己的观点/作品希望得到评价时

**结论：** 81% 的用户感知到 AI 讨好行为，验证了 Mind Lens 的核心立项假设。

### 2.7 信任建立的关键因素

## 四、功能需求

### 4.1 P0 — 核心功能（MVP 必须）

#### F1：多轮对话追问 + 渐进式卡片更新

**用户故事：** 作为一个纠结中的用户，我经常描述不清自己的困境，希望 AI 能主动引导我说出关键信息；同时希望看到分析在每轮对话后逐步完善的过程。

**交互设计：**

前端保留五维卡片布局（表层问题 / 隐含假设 / 信息缺失 / 认知盲点 / 建议），但交互逻辑改为**多轮渐进式更新**：

1. 用户输入问题后，Agent 判断信息是否充分
2. 若信息不足，返回追问 + 前 4 张卡片的当前分析（第 5 张"建议"锁定）
3. 用户回答追问后，Agent 更新前 4 张卡片内容（每轮直接覆盖）
4. 当 Agent 认为信息足够时，询问用户"是否查看建议"
5. 用户同意 或 达到最大轮次（**5 轮**）→ 解锁第 5 张"建议"卡片

**卡片状态流转：**

```
初始状态:  [表层...] [假设...] [缺失...] [盲点...] [建议🔒]
  ↓ 每轮追问后
更新状态:  [表层██] [假设██] [缺失██] [盲点░░] [建议🔒]
  ↓ 信息充分 & 用户同意 / 达到5轮
最终状态:  [表层██] [假设██] [缺失██] [盲点██] [建议✅]
```

**LLM 输出格式约定（JSON）：**

```json
{
  "cards": {
    "surface": "表层问题分析文本（或null）",
    "assumption": "隐含假设分析文本（或null）",
    "info_gap": "信息缺失分析文本（或null）",
    "blind_spot": "认知盲点分析文本（或null）",
    "suggestion": "建议文本（仅在ready时非null）"
  },
  "follow_up": "追问内容（如无追问则为null）",
  "ready_for_suggestion": false,
  "round": 1
}
```

**卡片更新策略：** 每轮直接覆盖，不做增量标记。

**技术方案：** 暂不接入 RAG 向量库，将心理学框架内容直接嵌入 system prompt（prompt stuffing），优先验证多轮对话机制的有效性。

**验收标准：**
- 信息充分的问题可直接输出完整分析
- 信息模糊的问题至少触发 1 次追问
- 追问不超过 5 轮
- 每轮追问后前 4 张卡片内容更新
- 第 5 张"建议"仅在条件满足时展示

**问卷依据：**
- 38% 用户选择"像剥洋葱一样逐层深入"
- 31% 用户选择"像聊天一样 AI 问一个我答一个"
- 75% 的痛点是"信息不全不敢决定"

---

#### F3：RAG 知识库接入

**用户故事：** 作为用户，我希望 AI 的分析有专业心理学框架的支撑，而不是泛泛而谈。

**需求描述：**
- 将 `rag/` 目录下的框架、模板和案例接入向量检索
- 使用轻量向量库（如 ChromaDB）进行语义检索
- 每次分析前，检索最相关的分析框架和案例
- 将检索结果注入 system prompt，增强分析质量

**验收标准：**
- 接入 RAG 后的分析质量明显优于纯 prompt 版本（A/B 测试评分提升 ≥15%）
- 检索延迟 < 500ms

**问卷依据：** 63% 用户"非常希望"有明确分析框架，*"要专业，言之有物"*。

---

#### F4：移动端响应式适配

**用户故事：** 作为手机用户，我希望在手机上也能舒适地使用 Mind Lens。

**需求描述：**
- 输入页面：全屏文本区，大按钮，底部固定
- 结果页面：单列卡片流，适配各种屏幕宽度
- 触摸友好：点击区域 ≥ 44px，卡片间距合理
- 底部导航栏（移动端）

**验收标准：**
- 在 375px 宽度下完整可用
- Lighthouse 移动端评分 ≥ 80

**问卷依据：** 81% 的问卷填写者来自手机端。

### 4.2 P1 — 重要功能（第二迭代）

#### F5：思维轨迹与成长记录

**用户故事：** 作为长期用户，我希望能回顾过去的困惑和分析，看到自己的成长与思维变化。

**需求描述：**
- 用户账户系统（邮箱或微信登录）
- 每次分析结果自动保存为一条"思维记录"
- 时间线视图：按时间展示所有分析记录
- 支持标签分类（职场、情感、规划等）
- 月度/季度"思维复盘"摘要（AI 自动生成）

**验收标准：**
- 用户可查看历史所有分析记录
- 时间线 UI 流畅可滑动
- 标签筛选功能正常

**问卷依据：** "保存思维轨迹"是用户信任的第一因素（44%），多位用户在开放题中明确提到。

#### F6：分析结果导出

**用户故事：** 作为用户，我希望能把分析结果保存或分享给信任的人。

**需求描述：**
- 一键导出为长图（适合社交分享）
- 导出为 Markdown / PDF 文件
- 复制到剪贴板

**验收标准：**
- 导出的内容格式美观、完整
- 长图包含品牌标识

**问卷依据：** 用户原声：*"分析结果等能一键导出"*。

#### F7：反讨好增强——立场坚守机制

**用户故事：** 作为用户，我不信任那些"听我抱怨就顺着说"的 AI，我需要一个敢于坚持观点的分析工具。

**需求描述：**
- 当用户对分析表示不满/反驳时，AI 不简单改口
- 区分"用户提供了新信息"（应更新分析）和"用户只是不认同"（应坚持并解释）
- 在 prompt 中加入"立场坚守"指令
- 追加反讨好检测：如果 LLM 输出包含"你说得对"、"确实如你所说"等讨好模式，触发重新生成

**验收标准：**
- 用户仅表达不满（未提供新信息），AI 坚持分析并补充论据
- 用户提供新信息，AI 合理调整分析
- 讨好模式检测覆盖率 ≥ 80%

**问卷依据：** 50% 用户明显感觉 AI 在讨好；最常见触发场景是"表示不满后 AI 改口"。

---

#### F8：流式输出优化

**用户故事：** 作为用户，等待分析结果时的空白期让我焦虑，希望能看到 AI 正在"思考"的过程。

**需求描述：**
- 结果页启用 SSE 流式输出（当前已有后端支持，前端未使用）
- 逐字/逐句渲染分析内容
- 显示"正在分析第 N 层..."的进度提示
- 打字机效果的文字出现动画

**验收标准：**
- 首字符出现时间 < 2 秒
- 流式渲染无卡顿

## 五、非功能需求

### 5.1 性能

| 指标 | 目标值 |
|------|--------|
| 首页加载 | < 1.5 秒（移动端 4G） |
| API 首字响应 | < 2 秒 |
| 完整分析生成 | < 15 秒 |
| RAG 检索延迟 | < 500ms |

### 5.2 隐私与安全

- 用户对话数据加密存储
- 不将用户原始对话用于模型训练
- 提供"删除我的数据"功能
- 隐私政策页面（用户在问卷中明确提到隐私顾虑）

### 5.3 可用性

- 降低使用门槛："太麻烦"是第一流失原因
- 首次使用引导（onboarding）不超过 3 步
- 无需注册即可体验核心功能（注册解锁历史记录等高级功能）

## 六、技术方案建议

### 6.1 核心架构决策：心理学框架的接入方式

针对"如何为 Agent 提供心理学分析框架"这一核心问题，有三种方案：

| 方案 | 做法 | 质量 | 延迟 | 可控性 | 开发成本 |
|------|------|------|------|--------|---------|
| **A. Prompt 固定框架** | system prompt 中写死理论 | 单一视角，覆盖面窄 | 最低 | 最高 | 最低 |
| **B. Agent 搜索工具** | Agent 自主决定搜什么理论 | 灵活但搜索质量不可控 | 最高 | 最低 | 最高 |
| **C. 多框架 RAG + 语义匹配（推荐）** | 向量库预存多框架，自动匹配 | 最佳平衡 | 中等 | 高 | 中等 |

**推荐路线：方案 C，通过 Function Calling + 内部向量库实现。**

Agent 拥有 `search_framework` 工具，可以自主决定是否调用向量库检索心理学框架。知识库内容由团队人工策划、审核，质量可控；语义匹配确保不同类型的问题命中最适合的理论。

**Agent 循环流程：**

```
用户输入
  ↓
Agent 第一轮：分析问题 → 决定是否调用 search_framework
  ├── 调用 → 向量库返回 top-2~3 相关框架
  │           ↓
  │         Agent 第二轮：基于框架 + 用户问题 → 输出分析
  └── 不调用 → 直接输出分析（简单问题）
```

**知识库结构设计：**

```

├── frameworks/              # 分析框架（按理论分文件）
```

### 6.2 短期实施（基于现有架构）

| 方向 | 方案 |

| 流式输出 | 前端接入已有 SSE 端点 |

## 八、成功指标

| 指标 | 目标 | 说明 |
|------|------|------|
| 首次分析完成率 | ≥ 70% | 用户输入问题后完整看完分析 |
| 次日留存率 | ≥ 30% | MVP 阶段 |
| 平均分析深度 | ≥ 3 层 | 剥洋葱交互中用户主动点击深入的层数 |
| 用户满意度 | ≥ 4.0/5 | 分析完成后评分 |
| 反讨好有效率 | ≥ 80% | 面对反驳不改口的比例 |
